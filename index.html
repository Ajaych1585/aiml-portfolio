AIML Study Assistant Bot

AI LAB · GUIDED ACTIVITY · WORKSHOP 1 · JULY 2026

Introduction

A browser-based AI study assistant built during the AIML-500 AI Lab, designed to help beginner students connect AI/ML definitions to real examples and quiz-style practice — the gap I noticed most first-time learners fall into.

Description

The assistant explains core AI/ML concepts (AI vs. ML, supervised/unsupervised/reinforcement learning, neural networks), builds short study plans on request, generates quiz-style practice questions with explanations, and covers responsible-AI topics like bias and fairness. It also deliberately redirects "do my assignment for me" requests back toward learning rather than answering them outright.

Live Prototype — try asking it something
Objective

To practice using generative AI tools and a structured design-thinking process to build a real, working AI product — and to give beginner AIML-500 students a low-stakes way to check their own understanding of course concepts before quizzes.

I started by testing the same study-plan prompt across three separate AI chats to compare organization and depth, then practiced iterating on a GPT-style assistant — asking it to revise, simplify, justify its structure, and handle a "what if" scenario. That iteration process directly shaped the bot's tone and scope before I wrote a single line of code. I then built the prototype in vanilla HTML/CSS/JavaScript and tested it against six scenarios (concept explanations, quiz generation, ethics questions, and an out-of-scope request) — all passed, after which I added quick- reply buttons and a fallback response for unrecognized questions.

Value Proposition

This artifact shows I can take an ambiguous brief ("build an AI assistant"), scope it to a real user need through design thinking, and ship a working, tested product — not just a written plan. It also shows I treat responsible AI use as a design requirement, not an afterthought.

Unique Value

Unlike a purely academic exercise, I approached this with the habits of a working full-stack engineer: define the scope, test against concrete scenarios, and iterate based on results. Coming from banking, healthcare, and finance — industries where an AI tool that gives a wrong or unsafe answer has real consequences — I built in guardrails (no private data, no completed-assignment answers) by default rather than adding them only after a problem surfaced.

Relevance

Directly relevant to where I'm headed professionally: applying AI/ML capabilities inside the kind of enterprise, regulated systems I already build as a Java full-stack engineer. This project is a small-scale proof that I can identify a user problem, apply responsible-AI thinking, and deliver a working tool — the same skill set, at a larger scale, that banking and healthcare employers need from AI-literate engineers.
